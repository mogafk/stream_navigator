### Stream navigator

Позволяет настраивать взаимодействия чата на твиче с компьютером стримера. Например добавить команду `!180` и его персонаж совершит резкий разворот на 180 градусов или на команду `!стан` блокировать управление клавиатурой и мышкой.

### Config

```
{
  // Ссылка на твой твчи канал, где будут слушаться сообщения
  "twitchLink": "https://www.twitch.tv/<your_channel_here>",
  // Возможность отключить звук событий
  "mute": false,
  "features": [
    {
      // Тип события
      "feature": "stun",
      // Команда в чате, по которой будет происходить событие
      // может содержать несколько комманда ["!комманд1", "!комманда2"]
      "chatCommand": ["!стан"],
      // Команда из чата будет выполняться не чаще чем cooldown
      // не действует на назначенный debugKey
      "cooldown": "1m",
      // Кнопка для дебага, по которой будет тригерится
      // может быть пустым
      "debugKey": "F2",
      // Время стана
      "time": "10s"
    },
    {
      // Тип события
      "feature": "turn",
      // Тоже что для feature = "stun"
      "chatCommand": ["!180"],
      // Тоже что для feature = "stun"
      "cooldown": "30s",
      // Тоже что для feature = "stun"
      "debugKey": "F3",
      // Расстояние поворота камеры. Зависит от DPI
      "distance": 370,
      // Зажимать ли мышь при повороте камеры
      // В играх вроде WoW для поворота камеры при движении мышки необходимо
      // зажимать кнопку мыши
      // Возможные значеия SMB(Secondary mouse button) и PMB(Primary mouse button)
      "modificatorKey": "SMB"
    }
  ]
}
```

Пустые значения `debug180Key` и `debugStunKey` просто выключают тригер этих событий по кнопке, команды из чата буду работать по прежнему.

### Сборка

```
go build -o stream_navigator64win.exe ./src/
```

### Известные проблемы

Если запускается не от имени администратора, то при выполнении команды stun мышкой можно двигать(клики работать не будут).
